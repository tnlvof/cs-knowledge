### Round 1 오답 노트

**Q2. GET vs POST**
- 내 답변: 데이터 노출 여부 차이만 언급
- 정답 핵심: 용도의 차이가 본질 (GET = 조회, POST = 생성/수정). 그 외 캐싱 가능 여부, 북마크 가능 여부, 길이 제한 등도 차이점.

**Q3. 프로세스 vs 프로그램**
- 내 답변: 프로세스로 프로덕트를 만드는 게 프로그램 (반대로 이해)
- 정답: 프로그램 = 디스크에 저장된 정적인 실행 파일 (코드 덩어리). 프로세스 = 프로그램이 메모리에 올라가 실행 중인 인스턴스. 비유: 프로그램은 레시피, 프로세스는 레시피로 실제 요리하는 행위.

**Q5. HTTPS 보안**
- 내 답변: 데이터 노출 방지만 언급
- 정답 핵심: HTTPS는 3가지 보안 제공 — (1) 기밀성: TLS 암호화로 도청 방지, (2) 무결성: 전송 중 데이터 변조 감지, (3) 인증: 인증서로 서버 신원 확인

**Round 2 합계: 3.5/5**

### Round 2 오답 노트

**Q6. Docker vs VM**
- 내 답변: Docker는 패키징, VM은 OS
- 정답 핵심: 커널 공유 여부. Docker는 Host OS 커널을 공유하고 프로세스만 격리 (가볍고 빠름, MB/초 단위). VM은 각각 완전한 OS 포함 (무겁고 느림, GB/분 단위).

**Q7. Metrics/Logs/Traces**
- 내 답변: 트레이스 모름, 로그부터 봐야 한다고 추측
- 정답 핵심: 순서는 메트릭 먼저 → 로그 → 트레이스. 메트릭으로 "무엇이 문제인가", 로그로 "왜 문제인가", 트레이스로 "어디서 문제인가" (분산 시스템에서 요청 경로 추적).

**Q10. kill -9 vs -15**
- 내 답변: -9 강제종료만 알고 있음
- 정답 핵심: SIGTERM(-15)은 정상 종료 요청으로, 프로세스가 cleanup(파일 저장, 커넥션 정리) 후 스스로 종료. SIGKILL(-9)은 즉시 강제 종료로 cleanup 없음. 실무 원칙: 항상 -15 먼저, 안 되면 -9.
